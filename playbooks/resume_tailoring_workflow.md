# Resume Tailoring Workflow

这个流程用于根据目标 JD 调整简历、项目描述和面试口径，同时保持事实准确。

## 输入

- `agent_knowledge/candidate_profile.json`
- `agent_knowledge/resume_summary.md`
- `agent_knowledge/resume_material_mapping.md`
- `inputs/jd/` 下的目标 JD 文件
- 可选：历史版本简历或目标公司面试侧重点

## 输出

建议输出到 `output/resume_tasks/`，每次至少包含：

- JD 要求拆解
- 你的经历与 JD 的映射
- 可以增强的简历点
- 不能硬写的能力边界
- 改写后的项目描述或简历 bullet
- 面试时的补充表述建议

## 执行步骤

1. 提取 JD 关键词  
   先拆出岗位最关心的：
   - 核心技术栈
   - 业务方向
   - 能力关键词
   - 加分项

2. 建立证据映射  
   按 `强支撑 / 中支撑 / 低支撑` 归类：
   - 强支撑：可以直接进入简历主表述
   - 中支撑：可以在面试中补充，不建议硬写成主成果
   - 低支撑：只能说了解，不进简历核心卖点

3. 优先改写主项目  
   优先围绕主项目做贴合，而不是平铺大量课程经历。  
   推荐顺序：
   - 业务目标
   - 关键技术路线
   - 你的职责
   - 指标和结果

4. 输出版本化建议  
   至少给出：
   - 保留原句
   - 建议增强句
   - 建议删除或降级句

5. 补上面试口径  
   对每个改写点，补一句“面试时可以怎么展开”，避免简历写了但答不稳。

## 判断标准

- 改写后是否更贴合目标岗位
- 是否仍然完全可由你的真实经历支撑
- 是否能在面试中展开到实现细节
- 是否避免课程项目越界替代工作项目

## 禁止项

- 不要为了贴 JD 生造不真实经历
- 不要把课程项目当成工作主成果
- 不要写出无法在 2 分钟内展开说明的 bullet
