# agent资料 说明

`agent资料/` 是原始课程材料、实验项目、示例代码和训练副产物的混合目录。

这个目录在本地仍然有保留价值，但 GitHub 仓库不需要把原始大文件一起带上去。当前处理方式是：

- 保留目录结构、源码、Markdown、配置和轻量文档
- 排除模型权重、视频、语料缓存、向量库卷数据、训练检查点和超大 Notebook
- 用索引文件记录这些被排除内容的路径、体积和用途

## 需要关注的索引文件

- `LARGE_FILE_INDEX.md`: 适合人读的大文件索引
- `large_files_manifest.json`: 适合脚本消费的大文件清单

## 目录角色

- `S2-系统训练营/`: 课程实操和训练资料，体积最大
- `S3-急救营/`: SFT、LoRA、RM 训练相关材料
- `S4-agent开发/`: Agent、RAG、训练项目和 Notebook

## 入 Git 原则

- 优先入 Git：文档、源码、说明、轻量实验样例
- 不入 Git：大文件原件、缓存、可重新下载或重新生成的运行产物
- 不确定时：先看 `LARGE_FILE_INDEX.md`，再决定是否需要保留索引而不是保留原件
