# Answer Generation Rules

这个文件的目标是约束后续回答：

- 要像你会说的话
- 要尽量使用你已经学过的知识
- 不要为了“高级感”引入陌生表述

生成回答前，建议先看一遍 `sample_answers.md`，默认沿用其中的句式密度和术语选择。

## 0. 面试现场口语约束

后续生成的回答，默认要满足“可以直接读给面试官听”。

### 语气要求

- 像真实候选人回答，不像文档总结
- 专业，但不要书面腔太重
- 先正面回答问题，再展开
- 句子尽量短，不堆太多并列概念

### 开场方式

优先用这类开场：

- `我理解这个项目核心解决的是……`
- `这个问题我会从两层来回答……`
- `如果结合我的项目，我会这样理解……`

少用这类开场：

- `这个项目核心不是……而是……`
- `你也可以把它理解成……`
- `本项目旨在……`

### 结构要求

- 口语回答优先 2 到 4 段
- 每段只讲一个重点
- 如果点比较多，优先说 `我简单分三点说`
- 结尾最好有一句收束，说明为什么这样设计

### 用词要求

优先使用：

- `我当时`
- `我们当时`
- `这个场景里`
- `实际遇到的问题是`
- `所以后面我们才`

少用：

- `本质上`
- `范式`
- `赋能`
- `抓手`
- `沉淀为闭环能力`

## 1. 回答生成总原则

### 原则一：先项目，后原理

回答时优先顺序：

1. 我在项目里怎么做
2. 这个做法对应什么原理
3. 为什么这样做而不是别的
4. 效果或结果

### 原则二：先熟悉词，再高级词

优先使用你学过的课程词汇：

- `混合检索` 优先于 `heterogeneous retrieval`
- `重排` 优先于 `re-ranking stage`
- `工具调用` 优先于 `tool invocation policy optimization`
- `反思纠错` 优先于 `self-corrective iterative reasoning`

如果要用高级词，必须先说中文直白解释。

### 原则三：回答里必须有“落地感”

不要只讲定义，要带至少一个：

- 场景
- 失败模式
- 权衡
- 指标

## 2. 项目类问题的回答模板

### 模板 A：项目背景题

适用问题：

- 你的项目解决什么问题？
- 为什么要做这个平台？
- 为什么不用普通问答系统？

回答顺序：

1. 业务痛点
2. 为什么普通 LLM / 普通 RAG 不够
3. 你们的系统是怎么拆的
4. 结果

### 模板 B：方案设计题

适用问题：

- RAG 知识层怎么设计？
- Agent 工作流怎么编排？
- Tool Schema 怎么抽象？

回答顺序：

1. 总体链路
2. 每一层职责
3. 核心 trade-off
4. 你负责的部分

### 模板 C：优化题

适用问题：

- 为什么做混合检索？
- 为什么做重排？
- 为什么 Query Rewrite 有用？

回答顺序：

1. 原方案问题
2. 新方案机制
3. 代价
4. 指标变化

### 模板 D：训练题

适用问题：

- 为什么做 SFT？
- LoRA 为什么适合？
- GRPO 奖励怎么设计？

回答顺序：

1. 为什么 Prompt 不够
2. 训练目标是什么
3. 数据 / 奖励怎么构造
4. 如何评估

## 4. 回答必须避免的 5 类问题

### 1. 不要把课程 Demo 说成真实线上系统

错误示例：

- `我们线上 Multi-Agent 调度就是这样做的`

更安全的说法：

- `我在课程项目和 Demo 里系统实践过这种调度方式，如果迁移到生产环境，我会这样设计`

### 2. 不要捏造没有材料支撑的数字

例如：

- QPS
- GPU 卡型
- 样本量
- 训练时长
- 线上调用量

### 3. 不要把知识点答得太“论文腔”

如果用户问的是工作项目，不要上来就讲：

- `policy gradient objective`
- `credit assignment`
- `hierarchical planner abstraction`

除非后面被追问。

### 4. 不要引入你没学过的方案当主答案

例如：

- ReWOO
- Tree of Thoughts
- DSPy 优化链
- Toolformer

这些可以作为补充，不应作为主答案，除非 ready 或原始资料明确覆盖。

### 5. 不要忘记回到简历

每个回答至少要有一句落回你的主项目：

- `在我的测试 Agent 项目里，这个问题主要体现在……`
- `我当时之所以关注这个点，是因为……`

---

## 5. 让回答更“像你学过”的具体要求

### 回答长度

- 单题口语回答：200-400 字优先
- 深挖题：400-700 字
- 一面场景优先控制在 45 到 90 秒可讲完

### 结构偏好

- 少用长定义
- 多用“问题 -> 方案 -> 结果”
- 默认让第一句就是明确结论，不要绕

### 词汇偏好

优先：

- `链路`
- `召回`
- `重排`
- `过滤`
- `评估`
- `闭环`
- `稳定性`
- `参数抽取`
- `链式调用`

少用：

- `范式转移`
- `认知架构`
- `策略梯度优化目标`

---

## 6. 后续生成回答时的硬约束

每次回答至少满足以下 4 条中的 3 条：

- 明确对应到简历主项目
- 使用至少 2 个 ready 中出现过的熟悉概念
- 有至少 1 个失败模式或 trade-off
- 有至少 1 个结果、指标或收益
